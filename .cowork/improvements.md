# 単語カードアプリ — 改善点・機能追加アイデア

> 作成日: 2026-04-06

---

## 1. 学習効率の向上

### 1-1. 習熟度トラッキング（優先度：高）
現状は「次へ / 前へ」で順番に進むだけで、正解・不正解の記録が存在しない。カードをめくった後に「わかった / わからなかった」ボタンを追加し、スコアを localStorage または Supabase の DB テーブルに保存することで、苦手カードを重点的に出題できるようになる。

### 1-2. 間隔反復（SRS）アルゴリズム
SM-2 などのアルゴリズムを導入し、習熟度に応じて次回出題タイミングを自動調整する。毎日起動したときに「今日やるべきカード」だけが表示されるモードを作れると学習効率が大幅に向上する。

### 1-3. 学習進捗サマリー画面
ファイル一覧の横に「最終学習日」「習熟率（%）」「連続学習日数」などを表示するダッシュボードを設ける。

---

## 2. テストモードの拡充

### 2-1. 4択クイズモード
現状のテストモードは表裏を反転するだけ。同一CSVの他カードからランダムに誤答選択肢を生成する4択形式を追加すると、受動的なめくり学習より記憶定着が高まる。

### 2-2. タイピング入力モード
裏面テキストをキーボードで入力させ、正誤判定（完全一致 / ひらがな正規化など）を行うモード。英単語スペルの練習などに有効。

### 2-3. タイム制限モード
1枚あたり N 秒のカウントダウンを表示し、時間内に答えられなかったカードは自動的に「不正解」扱いにする。

---

## 3. データ管理・共有

### 3-1. フォルダ / タグ管理
CSVが増えてきたときのために、フォルダ階層またはタグによる分類機能を追加する。Supabase Storage のパスプレフィックス（例: `英語/vocabulary_001.csv`）を利用することで最小限の改修で実現できる。

### 3-2. CSVのダウンロード・エクスポート
アップロード済みCSVをワンクリックでローカルに保存できるボタンを追加する。バックアップや他アプリへの転用に役立つ。

### 3-3. カードのインライン編集
ファイル一覧から各カードを確認・修正できる編集モーダルを追加する。誤字や訳の修正のたびにCSVを再アップロードする手間がなくなる。

### 3-4. 共有リンク機能
Supabase の Public URL を使い、特定ファイルをURLで共有できる機能。グループ学習や教材配布に便利。

---

## 4. UX・インターフェース改善

### 4-1. ダークモード対応
`prefers-color-scheme: dark` メディアクエリ + CSS カスタムプロパティを使ったテーマ切り替え。長時間学習時の目への負担を軽減する。

### 4-2. スワイプジェスチャー対応
スマートフォンでの利用時に左右スワイプでカード送り、上下スワイプでめくり操作ができるとタッチ操作がより自然になる（Touch Events API で実装可能）。

### 4-3. カードのフォントサイズ自動調整
長いテキストがカードからはみ出す問題を、`font-size` を動的に縮小することで解決する。

### 4-4. 音声読み上げ（TTS）
Web Speech API の `SpeechSynthesis` を使い、表面・裏面のテキストを自動読み上げするボタンを追加する。英語リスニング学習に有効。

---

## 5. PWA・オフライン対応

### 5-1. Service Worker によるオフラインキャッシュ
現状 `manifest.webmanifest` はあるが Service Worker が未実装。`Workbox` などを使いアプリシェルをキャッシュすることで、ネットワーク不在時でも起動できるようにする。

### 5-2. ローカルストレージへのCSV保存オプション
Supabase に加えて、`IndexedDB` にCSVを保存するオプションを設けることで、アカウント不要のオフライン専用モードを実現できる。インターネット環境がない場所での学習に対応できる。

---

## 6. セキュリティ・設定まわり

### 6-1. 設定画面の UI 化
現状は `js/config.js` を直接編集する必要があり、非エンジニアにはハードルが高い。アプリ内に「設定」モーダルを設け、URL / Key を入力して `localStorage` に保存する仕組みにすることで、コード変更なしにデプロイ済みアプリを利用できるようになる。

### 6-2. Supabase Row Level Security の徹底
現状は anon ユーザーに読み書きを許可する設定が前提だが、認証（Supabase Auth）を追加してユーザーごとにCSVを分離することで、複数人での利用時にもプライバシーを保てる。

---

## 7. その他・将来的なアイデア

- **多言語対応**: UI テキストを i18n 化し、英語・中国語など他言語ユーザーにも対応
- **画像カード**: CSVの3列目に画像URLを指定して画像付きカードを表示
- **AI自動生成**: OpenAI API などと連携し、単語リストから例文や解説を自動生成
- **学習カレンダー**: GitHub の草（Contribution Graph）風に学習記録をカレンダー表示
- **複数ファイルのマージ学習**: 複数CSVを選択して合算したカードセットで学習する機能
